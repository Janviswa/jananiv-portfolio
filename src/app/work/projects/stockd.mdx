## Overview

Stockd is a production-grade inventory reservation system designed to manage stock across multiple warehouses while preventing overselling during concurrent transactions. The platform provides real-time inventory tracking, intelligent reservation management, geo-aware warehouse allocation, and automated stock recovery mechanisms, ensuring accurate inventory control and operational reliability.

## Problem

Modern inventory systems must handle thousands of simultaneous reservation requests without creating stock inconsistencies. Traditional approaches often suffer from race conditions, duplicate reservations, overselling, and inefficient warehouse allocation. There is a need for a scalable reservation platform capable of maintaining inventory accuracy under high concurrency.

## Solution

Stockd addresses these challenges through a multi-warehouse reservation system built on PostgreSQL transactions, Redis-based idempotency, and real-time inventory synchronization. The platform uses row-level database locking and serializable transactions to guarantee that concurrent reservation requests are processed safely. Automated reservation expiry, stock release mechanisms, and geo-aware routing further improve inventory availability and operational efficiency.

## Key Features

- **Multi-Warehouse Inventory Management** — Tracks inventory across multiple warehouse locations.
- **Inventory Reservation System** — Reserve, confirm, release, and manage stock reservations.
- **Race Condition Prevention** — Uses PostgreSQL row-level locking and serializable transactions to prevent overselling.
- **Idempotent API Operations** — Prevents duplicate reservations and confirmations using Redis-based idempotency keys.
- **Automatic Reservation Expiry** — Releases expired reservations and restores stock availability.
- **Geo-Aware Routing** — Allocates inventory from the most suitable warehouse location.
- **Real-Time Dashboard** — Monitor inventory levels, reservations, warehouse status, and system activity.
- **Concurrency Simulation** — Interactive demonstration of concurrent reservation handling and stock locking.
- **Audit Logging** — Tracks reservation lifecycle events and inventory actions.

## Challenges and Learnings

The primary challenge was preventing race conditions when multiple users attempted to reserve the same inventory simultaneously. Implementing PostgreSQL row-level locking, serializable transactions, and Redis-based idempotency required careful system design to guarantee data consistency. The project strengthened skills in distributed systems, concurrency control, database transactions, caching strategies, and scalable backend architecture.

## Outcome

Stockd successfully delivers a reliable inventory reservation platform capable of handling concurrent requests without overselling or duplicate transactions. The system demonstrates production-grade reservation management, automated inventory recovery, and real-time stock visibility, showcasing practical expertise in full-stack development, database engineering, and scalable system design.

## Key Learnings

This project provided hands-on experience in transaction management, concurrency control, idempotent API design, Redis caching, database optimization, and production-scale application architecture. It also strengthened understanding of inventory management workflows and distributed system reliability.
